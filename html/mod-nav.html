MyAMS "nav" module

MyAMS's nav module is used to handle navigation into the web application. It defines navigation menus creation, and handles several kinds of links as already explained into Links and navigation with MyAMS.

Navigation menus

Navigation menus are an important part of any web application; a page is dedicated to them in the Layout section (see Navigation menus), where you can discover how to create them.

Module API

class NavigationMenu

This class is used internally by MyAMS to create menus and register events handlers. But the class is exported so that you can use it in your own code if required.

NavigationMenu.constructor(menus, parent, settings)

The constructor receives three arguments which are the menus definition object, the parent into which menus are to be added, and the component settings; definition object can be null if menus definition is already included into HTML code. See Navigation menus to know how menus are defined.

Settings is an object which can provide several attributes: accordion is a boolean value specifying if menu should be used in accordion mode or not, and is true unless explicitly defined to false; speed is a number which specifies how quickly the menu items are opened or closed; default value is 200 and is given in milliseconds.

NavigationMenu.getMenus()

This method is used to create HTML elements from menus definition provided to the constructor, using internal Menu and MenuHeader classes.

Please note that the method is creating and returns HTML elements, but without including them into the DOM.

NavigationMenu.init(menus)

This method receives menus as HTML elements created by the getMenus method. She goes through all menus items, adds aditional elements if required (like opening/closing signs on items with sub-menus), and sets CSS classes on active menus.

NavigationMenu.render()

This method is calling the getMenus() method to create menus, then init() to initialize them; the menus are then added to the DOM.

MyAMS.nav.init()

This function is called automatically by MyAMS after module loading. She creates a JQuery $().navigationMenu() components, which is using the NavigationMenu class.

If MyAMS.config.ajaxNav is set to true (which is the default), this module also registers events handlers which are used to handle all navigation links, including those which are created into MyAMS navigation menu.

MyAMS.nav.initElement(element)

This function is called automatically by MyAMS on content initialization. It's only action is actually to create a navigation menu from the nav HTML element.

MyAMS.nav.setActiveMenu(menu)

This function is used to activate the menu given as argument. This activation will highlight the menu, and the navigation bar will be scrolled if necessary so that the activated menu is made visible.

This function is generally called automatically by MyAMS on any menu click.

MyAMS.nav.drawBreadcrumbs()

As explained in the Breadcrumbs ribbon's page of MyAMS layout section, breadcrumbs are also handled automatically when navigation menus are used; the activated menu is used to draw breadcrumbs.

MyAMS.nav.minifyMenu(evt)

This function is used as a click event handler of navigation menu minifying button (see Navigation menus in MyAMS's layout section).

The navigation menu is minified by adding a minified class to the page body; the minifying state is then stored into local storage, and set automatically on page reload.

MyAMS.nav.switchMenu(evt)

This function is used as a click event handler of navigation menus show/hide button, located on the top right of the window (see Show/hide menus in MyAMS layout section).

The navigation menu is shown or hidden by adding a hidden-menu class to the page body; the state of the menus is stored into local storage, and set automatically on page reload.

MyAMS.nav.restoreState()

This function is used to restore state of navigation menus by looking into local storage.
