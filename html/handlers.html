MyAMS events handlers

Default events handlers

MyAMS events module is registering several events handlers which are bound to data-attributes.

These events can be bound to any HTML element, included elements included dynamically into the DOM.

Assigned callbacks can be defined in preloaded Javascript modules, or in extensions which are declared and downloaded dynamically (see modules).

  • click events handlers: you can set a data-ams-click-handler attribute on any HTML tag to declare a callback which will be called when a click event occurs on this element; you can declare several callbacks by separating their names with spaces or tabs.

    The callback will receive arguments this with the source element, the original click event, and additional parameters which can be defined as JSON in a data-ams-click-handler-options attribute.

    Please note that event.preventDefault() will be called except if you set data-ams-prevent-default or data-ams-click-prevent-default attributes to false, and that event.stopPropagation() will be called except if you set data-ams-stop-propagation or data-ams-click-stop-propagation to false.

    For any reason, you can also disable a click handler temporarily by setting a data-ams-disabled-handlers attribute to true, 'click' or 'all' values.

  • change events handlers: you can set a data-ams-change-handler attribute on any HTML input supporting "change" events to declare a callback which will be called when a change event occurs on the input; you can declare several callbacks by separating their names with spaces or tabs.

    The callback will receive arguments this with the source element, the original change event, and additional parameters which can be defined as JSON in a data-ams-change-handler-options attribute.

    Please note that event.preventDefault() will be called except if you set data-ams-prevent-default or data-ams-change-prevent-default attributes to false, and that event.stopPropagation() will be called except if you set data-ams-stop-propagation or data-ams-change-stop-propagation to false.

    For any reason, you can also disable a change handler temporarily by setting a data-ams-disabled-handlers attribute to true, 'change' or 'all' values.

  • reset callbacks: "reset" events handlers are associated with all forms, to reset alerts and validation warnings.

    You can also register your own additional callbacks by setting a data-ams-reset-callback on any element inside the form, which will be called on form reset with the form as this, and with additional parameters which are the element on which the callback was defined, and other options which can be defined as JSON in a data-ams-reset-callback-options attribute.

    "reset" events can also be handled using data-ams-reset-handler forms attribute; compared with reset callbacks defined just above, these handlers can prevent default reset by setting data-ams-keep-default or data-ams-reset-keep-default attributes to false; you can add additional parameters with data-ams-reset-handler-options attribute, in JSON format.

Custom events handlers

Besides standard events listed above, you can subscribe to any "custom" event using a data-ams-events-handlers data attribute applied on any HTML element; the value of this attribute is a serialized JSON object whose attributes names are the names of the events, and whose values are the names of the events handlers. For example, if you set data-ams-events-handlers='{"mouseover": "MyAMS.app.mouseHandler"}' on an HTML element, the given handler will be called every time a mouseover event will be triggered on the given element.

Generating click events

Instead of defining specific click handlers, it can be quite convenient in a few use cases to just trigger custom events when a click occurs on an HTML element; this can be done using a data-ams-click-event attribute, whose value is the name of the event which may be triggered; additional event options can be set using data-ams-click-event-options attribute.
