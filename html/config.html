MyAMS base configuration

MyAMS.config is a settings objet which contains configuration of several MyAMS parameters.

These settings can be changed or updated at will on MyAMS initialization.

  • ajaxNav: boolean value specifying if AJAX navigation, relying on several events handlers, is enabled or not; true by default; see Links and navigation with MyAMS to know more about how links and navigation are handled.

  • alertsContainerClass: this is the classname of the container used to handle alerts; toast-wrapper by default.

  • clearContent: string name of MyAMS function used to clear an HTML element before replacing it's content; MyAMS.core.clearContent by default.

  • csrfCookieName: this is the name of a cookie used to send CSRF token; crsf_token by default.

  • csrfHeaderName: this is the name of a HTTP header used to send CSRF token to server; X-CSRF-Token by default.

  • device: can be mobile or desktop according to detected platform.

  • enableTooltips: boolean value specifying if Bootstrap tooltips have to be enabled globally; true by default.

  • enableHtmlTooltips: boolean value specifying if HTML code is allowed into tooltips text; true by default. If your tooltips are generated with text which is not secure, you may disable this option to reduce risks of XSS attacks!

  • formChangeCallback: if forms changes warning are enabled globally using the warnOnFormChangeCallback, or for a specific form, MyAMS provides a generic callback which can also be replaced globally by giving it's name in this property...

  • initContent: string name of MyAMS function used to initialize any HTML content; MyAMS.core.initContent by default.

  • initPage: string name of MyAMS function used to initialize a full HTML page; MyAMS.core.initPage by default.

  • isMobile: boolean flag specifying if a mobile platform has been detected

  • menuSpeed: integer value used to set duration of navigation menu transitions, in milliseconds; 235ms by default.

  • modules: list of already loaded MyAMS modules .

  • safeMethods: this is a list of HTTP verbs which can be used without checking CSRF token; ['GET', 'HEAD', 'OPTIONS', 'TRACE'] by default.

  • useRegistry: this boolean option is used to specify if plugins registry should be used or not on contents initialization; value is true by default.

  • warnOnFormChange: boolean value specifying if an alert should be displayed when a user is leaving a form without saving modified information; this is true by default. The warning can be handled differently on specific forms, if required, using specific data attributes (see MyAMS forms options).
