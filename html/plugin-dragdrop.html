MyAMS "drap & drop" plug-ins

MyAMS drag & drop features are actually based on JQuery UI package. MyAMS is just a thin wrapper around JQuery UI plug-ins, and their documentation is the best source of information to know how they work.

A simple draggable element can be created like this:

	<div class="border border-danger width-100px height-100px mouse-move draggable"></div>

This is another draggable with a droppable:

	<div class="parent clearfix">
	  <div class="float-left border border-danger ml-3 mb-3 width-100px height-100px mouse-move draggable"
	     data-ams-draggable-containment="parent"
	     data-ams-draggable-helper="clone">
	  </div>
	  <div class="float-left border ml-3 mb-3 width-100px height-100px droppable"
	     data-ams-droppable-accept=".draggable"
	     data-ams-droppable-options='{
	       "classes": {
	         "ui-droppable-hover": "border-info"
	       }
	     }'>
	  </div>
	</div>

MyAMS draggable plug-in options

These plug-in options are available as data attributes prefixed by data-ams-draggable-:

Name Default Description
connectSortable false allows the draggable to be connected to the specified sortable, and to become part of it when the element is dropped; the helper option must be set to clone for this to work correctly; look at sortables plug-in to know more about sortables
containment false constrains dragging to the bounds of the element specified by this option; can be a JQuery selector, a DOM element, a custom string ("parent", "document" or "window") or a bounding box in the form of an array of coordinates [x1, y1, x2, y2]
cursor "move" mouse cursor to use when the element is moved
handle false if indicated, specifies a child element on which the mouse cursur must be clicked for the dragging to start
helper "original" allows for a helper element to be used when dragging; can be set to clone to drag a copy of the original element, or to a function which will return a DOM element used for dragging
options undefined serialized JSON object containing additional draggable settings; see JQuery-UI draggable documentation
start undefined name of a callable, called when dragging starts
stop undefined name of a callable, called when dragging stops

MyAMS droppable plug-in options

These plug-in options are available as data attributes prefixed by data-ams-droppable-:

Name Default Description
accept undefined selector or callable used to define which elements are accepted by the droppable
drop undefined name of a callable, called when an accepted draggable is dropped on the droppable
options undefined serialized JSON object containing additional droppable settings; see JQuery-UI droppable documentation
