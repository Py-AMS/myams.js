MyAMS "notifications" module

This small module is used to handle "notifications", which have been described into MyAMS's notifications page.

MyAMS.notifications.getNotifications(evt, options): Promise

This function is the default handler used to load and display notifications.
Notifications format API is described into notifications page given above.

By default, this handler is used as an event handler on show.bs.dropdown event:

	<div id="user-notifications" class="ml-1"
	   data-ams-events-handlers='{"show.bs.dropdown": "MyAMS.notifications.getNotifications"}'
	   data-ams-events-options='{"localTimestamp": "true"}'
	   data-ams-notifications-target="#notifications-pane"
	   data-ams-notifications-source="data/user-notifications.json">
	  ...
	</div>

The function relies on other modules; she returns a Promise which is resolved when ajax module is loaded and when all notifications have been loaded from remote URL.

Handler options and data attributes include:

  • ams-notifications-source: URL of the remote service providing notifications contents in JSON format;
  • ams-notifications-params: additional parameters, if required, used to get data from remote service;
  • ams-notifications-options: additional AJAX settings used by MyAMS.ajax.get to load remote data;
  • ams-notifications-target: JQuery selector of the DOM element where notifications may be inserted.

options is an object which can contain several attributes; if using MyAMS events handlers, it can be stored into ams-events-options data attribute:

  • localTimestamp: boolean flag used to specify if timestamp displayed below the notifications list is the one which is provided by the server (default, false value), or if a locally generated timestamp should be used (true value);
  • hideTimestamp: if false (default value), a timestamp is displayed next to each item in the notifications list, otherwise this information is hidden.
