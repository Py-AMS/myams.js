Form events

Several events are triggered during form submission:

  • init-data.myams.form: this event is triggered at the end of form's data initialization; if receives actual POST data and a veto object as arguments; if veto's veto attribute is set to true, the submit button is reset and the submit process is cancelled.

  • before-serialized.myams.form: this event is triggered before form fields serialization; it received a veto object argument: if it's veto attribute is set to true, the submit process is cancelled.

  • before-submit.myams.form: this event is triggered before form submit; current data is added as event argument, with a veto object: if it's veto attribute is set to true, the submit process is cancelled.

  • complete.myams.form: this event is triggered when submission is completed, with a success or an error status.

  • submit-success.myams.form: this event is triggered on successfull commit. It's arguments are the submit result, the request status, the request and a veto object; if veto.veto is set to true, the default MyAMS callbacks will not be called.

  • submit-error.myams.form: this event is triggered on failed commit.

  • after-submit.myams.form: this event is triggered when the form submit callback has been called.

  • after-reset.myams.form: this event is triggered when a non modal form has been submitted, and it's state reset.

Warning!

If you choose to cancel a submit process by using a veto object, it's up to you to reset form's state!
