Navigation menus

Navigation menus are displayed on the left side of the main window.

There are two ways to create navigation menus:

  • by using a static HTML definition;
  • by calling a remote service which will render menus definition in JSON format;
  • by using a mixed mode combining both of them.

The whole menu configuration and management is handled by MyAMS's nav module.

General rules

The navigation element is made of several "menus"; each menu is made of a header (by default with an orange background, and which can be empty) and of a set of menu entries; each menu entry contains a label, a link, an icon, a badge, and an optional set of sub-menus items; the link itself is optional for a menu containing sub-menus. There is no theorical limit in the depth of menus and sub-menus, but it's generally a better option to limit menus depth to a maximum of 3 levels.

Menus can be configured in accordion mode or not; if this mode is activated, all active menus are closed when another one is opened. To active this mode, set the data-ams-menu-accordion attribute to true on the nav element.

MyAMS provides it's own private factory defined into nav module; you can override this and set your own menus factory by setting a data-ams-menu-factory attribute with the name of this factory on the nav element.

Mixed mode

If required, you can mix a static menus definition with a JSON one; in this case, just create the HTML definition as usual and add the URL of the JSON definition file; menus created from JSON definition file will be placed after pre-created HTML menus.
